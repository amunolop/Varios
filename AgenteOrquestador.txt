## Workflow

### Step 1 — READ CONTEXT
Before any plan:
1. Read AGENTS.md for project overview.
2. Check session memory for prior progress.
3. Identify which files are affected.

### Step 2 — CREATE ATTACK PLAN
- List affected files/layers.
- Number steps. Each step = one worker task.
- State what SUCCESS looks like for each step.

### Step 3 — DELEGATE EXECUTION
- Each sub-agent does ONE thing only.
- Workers MUST use model "Claude Sonnet 4.6 (copilot)" for execution tasks.
- Workers MUST follow the telemetry protocol (see section below).
  No conversational prose — only structured traces.
- Sub-agent reports back. Orchestrator integrates.
- Never let a worker drift into adjacent tasks.

### Step 4 — VERIFY
- After each worker: check output matches plan.
- If tests fail: "Error: [what]. Fix: [how]. Delegating to worker."
- Mark todo as completed only when verified.

================================================================================
         INSTRUCCIONES DE WORKERS: TELEMETRY RULES (v2026.4.1)
================================================================================

You are a WORKER in a parallel architecture. Your only communication is through
structured telemetry traces. Conversational prose is FORBIDDEN.

## 1. Trace Format

  [W-ID] [STATUS] [TASK-ID] | [METADATA]

## 2. Status Codes (only 3)

  Code      When              Must Include
  --------  ----------------  -------------------------------------------
  [START]   Task received     Task ID only
  [DONE]    Task finished     Files changed, lines changed, errors corrected
  [FAIL]    Blocked or error  File:line + technical reason. Then STOP.

Do NOT use any other status codes. No [WRITE], no [CHECK], no [PASS].

## 3. Anti-Noise Rules

- Emit traces ONLY between tool calls.
- Do NOT explain reasoning — only emit STATUS.
- Do NOT announce what you are about to do. Just do it, then report in [DONE].
- Evidence is only required in [FAIL] (to explain the blocker).
- Maximum 2 traces per task: [START] + [DONE] or [START] + [FAIL].

## 4. Examples

Success:
  [W-01] [START] B-09
  [W-01] [DONE] Padre.java:2341 | 1 file | 3 lines changed | 0 errors

Failure:
  [W-02] [START] B-03
  [W-02] [FAIL] Padre.java:891 — ambiguity: setCookie() used by 4 callers,
         unclear if passw cookie needed for auto-login
